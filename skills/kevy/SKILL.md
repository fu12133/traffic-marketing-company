## 数据拉取与监控

### 拉取策略

```
定时任务: 每5分钟全账户扫描 | 每1分钟高消耗账户(>1w/日)

数据层级:
  Account Level (账户级)
  ├─ 账户消耗、余额、预算
  ├─ 账户日限额、月限额
  └─ 账户状态（正常/审核/暂停/封禁）

  Campaign Level (计划级)
  ├─ 计划消耗、展现、点击
  ├─ CTR、CPC、CPM、CPL
  ├─ 转化数、转化率、转化成本
  ├─ ROI、ROAS、GMV
  ├─ 出价、预算、花费
  └─ 状态（投放中/暂停/审核中/已结束）

  Ad Level (单元/创意级)
  ├─ 素材消耗、展现、点击
  ├─ 素材CTR、CVR
  ├─ 素材审核状态
  └─ 素材类型、尺寸、标签
```

### KPI 指标说明

| 指标 | 全称 | 计算公式 | 正常范围参考 |
|---|---|---|---|
| 消耗 | Spend | 当日累计花费 | 按预算 |
| CTR | Click-Through Rate | 点击量/展现量 × 100% | 信息流 0.5%-3% |
| CPC | Cost Per Click | 消耗/点击量 | 行业差异大 |
| CPM | Cost Per Mille | 消耗/展现量 × 1000 | 行业差异大 |
| CPL | Cost Per Lead | 消耗/转化数 | 按ROI倒推 |
| CVR | Conversion Rate | 转化数/点击量 × 100% | 1%-15% |
| ROI | Return on Investment | GMV/消耗 | >1.2 及格 |
| ROAS | Return on Ad Spend | 收入/消耗 | 同ROI |
| CPA | Cost Per Action | 消耗/转化数 | 同CPL |

## 自动优化引擎

### 1. 低效计划关停

```
触发条件:
  ├─ 消耗 > 2×目标CPC 且 转化=0
  ├─ CTR < 0.1% 且 消耗 > 500
  ├─ CPL > 目标CPL × 1.5
  ├─ ROI连续2小时 < 0.5
  └─ 转化数 > 10 且 转化率 < 0.5%

处理:
  1. 暂停计划
  2. 记录关停原因
  3. 更新关停日志
  4. 推送关停通知

保护机制:
  ├─ 新计划保护期: 前2小时不自动关停
  ├─ 主力计划 (消耗TOP10) 降级为人工确认
  ├─ 日关停上限: 账户日关停 ≤ 20%
  └─ 操作延迟: 每次关停间隔 ≥ 2分钟
```

### 2. 优质计划复制

```
优质判断标准:
  ├─ ROI > 目标ROI × 1.2
  ├─ CTR > 行业均值 × 1.5
  └─ CPL < 目标CPL × 0.7

复制策略:
  1. 保持原计划配置 (出价/定向/素材/落地页)
  2. 调整出价: 原出价 × 1.0 (保守) / × 1.1 (激进)
  3. 细分定向: 原定向拆分为 2-3 个子定向
  4. 更新素材: 使用同批素材新组合
  5. 设置预算: 原预算 × 0.5-0.8 (测试预算)

保护机制:
  ├─ 同计划克隆上限: 同一优质计划 ≤ 3 条克隆
  ├─ 日克隆上限: 账户日新增 ≤ 10 条
  └─ 克隆后观察期: 2小时，不达标自动关停
```

### 3. 出价微调

```
出价调整逻辑:
  ├─ CPL > 目标CPL × 1.3 → 降价 5%-10%
  ├─ CPL < 目标CPL × 0.7 → 可提价 3%-5% (放量)
  ├─ 消耗速度快于预期 → 降价 5%
  ├─ 消耗速度慢于预期 → 提价 5%-10%
  ├─ 展现量不足 → 提价 5%-10% + 检查定向宽度
  └─ 转化成本稳定且ROI达标 → 维持

约束条件:
  ├─ 单次调整幅度 ≤ ±15%
  ├─ 每日调整频次 ≤ 3次
  ├─ 调整间隔 ≥ 2小时
  └─ 新计划首6小时不调出价
```

### 4. 分时投放启停

```
规则引擎:
  ├─ 历史时段转化率分析
  ├─ 各时段ROI表现
  └─ 行业时段规律

启用时段:
  规则: 历史同一时段 ROI > 目标ROI × 0.8
  或:   历史同一时段转化率 > 日均转化率 × 0.7

暂停时段:
  规则: 历史同一时段 ROI < 目标ROI × 0.3
  或:   消耗 > 日均消耗 × 0.2 且 转化=0

动态调整:
  ├─ 实时数据覆盖历史规则
  ├─ 大促期间: 全时段投放
  ├─ 新品测试期: 全时段投放前24h
  └─ 预算紧张: 优先ROI最高时段
```

## 账户/素材违规监控

### 账户状态监控

```
账户状态变更监控:
  ├─ 正常 → 审核中    → 推送: 素材审核中，无需操作
  ├─ 正常 → 已封禁    → 推送: 🔴 P0 紧急，需立即处理
  ├─ 正常 → 暂停      → 推送: 🟡 账户暂停，检查平台通知
  ├─ 正常 → 余额不足  → 推送: 🔵 余额不足，提醒续费
  └─ 审核中 → 正常    → 推送: 🟢 审核通过，检查投放状态

检查频率: 每5分钟扫描全部账户状态
```

### 素材审核状态监控

```
素材审核状态:
  ├─ 审核中 → 无操作
  ├─ 审核通过 → 检查是否已开始投放
  ├─ 审核拒绝 → 推送: 🟡 素材违规，附违规原因
  └─ 审核中 >24h → 推送: 🔵 超长审核，建议联系平台

素材健康度:
  指标: 素材CTR vs 账户平均CTR
  ├─ CTR > 账户均值 × 1.2  → 🟢 优质素材
  ├─ CTR 在 0.8-1.2倍      → 🟡 普通素材
  └─ CTR < 账户均值 × 0.8  → 🔴 低质素材，建议替换

素材疲劳度:
  指标: 素材累计消耗 / 素材生命周期
  规则: 同素材消耗连续3日下降 → 建议替换
  规则: 素材上线 > 7天且CTR下降 > 30% → 建议替换
```

```python
from apscheduler.triggers.cron import CronTrigger
from datetime import datetime

scheduler = BackgroundScheduler()

@scheduler.scheduled_job('interval', minutes=5, id='core_monitor')
def core_monitor():
    for platform in ['oceanengine', 'baidu', 'kuaishou', 'tencent']:
        accounts = get_accounts(platform)
        for account in accounts:
            # 1. 拉取数据

            # 5. 消息推送
            dispatch_alerts(alerts, account)
```
